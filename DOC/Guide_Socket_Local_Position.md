# 📍 가이드: 소켓 로컬 포지션 (Socket Local Position)이란?

프리셋 모듈 데이터베이스에서 **`Local Position`**은 **"이 방 안에서 문(Socket)이 어디에 있는가?"**를 나타내는 **상대 좌표**입니다.

---

## 1. 기준점 (Origin) 정하기
모든 방(모듈)은 **바둑판(Grid)** 위에 올려져 있다고 생각하세요.
이 바둑판의 **가장 왼쪽 아래 칸**이 바로 **`(0, 0)`**입니다.

*   **X축**: 오른쪽으로 갈수록 커짐 `(0 -> 1 -> 2 ...)`
*   **Y축**: 위쪽으로 갈수록 커짐 `(0 -> 1 -> 2 ...)`

---

## 2. 시각적 예시 (3x3 크기의 방)

가로 3칸, 세로 3칸짜리 **"중간 크기 방(3x3)"**을 예로 들어보겠습니다.

### 🖼️ 그림 설명
각 `[ ]`는 타일 한 칸을 의미합니다.

```
      (위쪽, North)
      
      Y=2  [2,0] [2,1] [2,2]  <-- 이 줄이 Y=2
      Y=1  [1,0] [1,1] [1,2]
      Y=0  [0,0] [0,1] [0,2]  <-- 이 줄이 Y=0 (바닥)
            ^     ^     ^
           X=0   X=1   X=2
```

### 🚪 소켓 위치 잡기 예제

#### **상황 A: 북쪽(North) 벽 한가운데에 문을 만들고 싶다!**
*   가장 윗줄은 **Y=2**입니다.
*   가로 가운데는 **X=1**입니다.
*   따라서 소켓의 `Local Position`은 **`(1, 2)`**가 됩니다.
    *   `Direction`은 `North`로 설정합니다.

#### **상황 B: 동쪽(East) 벽 맨 아래에 문을 만들고 싶다!**
*   가장 오른쪽 줄은 **X=2**입니다.
*   가장 아랫줄은 **Y=0**입니다.
*   따라서 소켓의 `Local Position`은 **`(2, 0)`**이 됩니다.
    *   `Direction`은 `East`로 설정합니다.

---

## 3. 실전: 다양한 모양의 방

### 1x3 복도 (일자형 세로 복도)
```
[ Y=2 ]  <-- 북쪽 소켓 위치: (0, 2)
[ Y=1 ]
[ Y=0 ]  <-- 남쪽 소켓 위치: (0, 0)
(X=0)
```

### 3x1 복도 (일자형 가로 복도)
```
[X=0] [X=1] [X=2]
 ^           ^
서쪽 소켓    동쪽 소켓
(0, 0)      (2, 0)
```

---

## ✅ 요약 (Cheat Sheet)

데이터 에셋(`UPresetModuleDatabase`)에 값을 넣을 때 이 규칙을 따르세요:

1.  종이에 모듈을 그립니다.
2.  왼쪽 아래 칸을 `(0,0)`이라고 씁니다.
3.  오른쪽으로 가면 X + 1, 위로 가면 Y + 1 입니다.
4.  문이 있는 칸의 좌표를 읽어서 `Local Position`에 적습니다.
5.  문이 **바라보는 방향(밖으로 나가는 방향)**을 `Direction`에 적습니다.
